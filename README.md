# guide-m-site

## 项目简介

`guide-m-site` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常用于展示特定主题或功能，彼此之间无强关联，适合作为静态内容的集合存储与访问。

本仓库不针对任何单个域名或具体网站，仅作为一个通用页面存放与分发的空间。

## 目录说明

```
guide-m-site/
├── index.html          # 仓库首页 / 目录页
├── pages/              # 存放各个独立 HTML 页面
│   ├── example-page-01.html
│   ├── example-page-02.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：每个 HTML 文件对应一个独立的页面，文件名应反映内容主题。
- `assets/`：存放页面共用的样式、脚本或图片资源，避免重复引用。

## 页面归档说明

- 所有页面均以静态 HTML 格式存放，可直接通过浏览器打开或部署后访问。
- 页面之间无依赖关系，可独立更新或删除。
- 建议每个页面在 `<head>` 中添加简短的元描述或标题，便于识别。

## 维护说明

- 新增页面：在 `pages/` 下创建新 HTML 文件，并确保引用资源路径正确。
- 更新页面：直接编辑对应文件，注意保持与其他页面及资源的兼容性。
- 删除页面：移除对应文件，并检查 `index.html` 或目录页中是否仍有相关链接。
- 资源管理：若页面使用独立资源，建议放在 `assets/` 子目录中，避免散落。

## 使用建议

- 可通过 GitHub Pages 或其他静态托管服务将本仓库部署为站点。
- 建议使用相对路径引用资源，以保持部署后的可移植性。
